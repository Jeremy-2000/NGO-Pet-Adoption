# Phase Plan

## Phase 1

Objectives: architecture, ERD, schema, folder structure, authentication.

Done: shared bootstrap, secure sessions, config, schema, installer, admin login, role checks, docs.

Testing: run installer, log in as admin, verify protected routes redirect or deny access.

## Phase 2

Objectives: shelter approval, listings, upload system.

Done: shelter applications, admin approval workflow, approval-gated listing create/edit, image validation, resizing, thumbnails.

Testing: register shelter, approve it, create listing, upload JPG/PNG/WebP, confirm public listing.

## Phase 3

Objectives: search, homepage visibility engine, shelter profiles.

Done: configurable visibility scoring, homepage promotion, paginated filters, public shelter directory and profiles.

Testing: search by keyword/filter/location, inspect visibility score display, open shelter profile.

## Phase 4

Objectives: inquiries, favorites, voting system.

Done: inquiry forms, rate limiting, audit logs, shelter inquiry review, session/user favorites, matchup voting.

Testing: submit inquiry, favorite animal, submit report, cast vote, verify shelter/admin views update.

## Phase 5

Objectives: analytics, optimization, hardening, launch preparation.

Done: dashboard metrics, activity log, upload optimization, security headers, deployment/security/maintenance docs.

Testing: review dashboard counts, run syntax checks, confirm production env/config values.
